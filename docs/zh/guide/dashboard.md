# 仪表盘

仪表盘是 MSM 登录后的默认页面，提供系统概览和实时监控信息，帮助你快速了解系统运行状态。

## 页面概览

仪表盘由多个可拖拽的信息卡片组成，默认展示以下模块：

- **设备信息** — 主机名、系统平台、运行时间、操作系统、架构
- **硬件信息** — CPU 型号、核心数、内存容量、硬盘容量及使用率
- **资源使用趋势** — CPU 和内存使用率的实时趋势图
- **实时速率** — 当前连接数、上传/下载速度的实时曲线
- **统计信息** — 系统运行时间、总上传/下载流量、CPU/内存使用率

## 设备信息卡片

显示当前运行 MSM 的设备基本信息：

| 字段 | 说明 |
|------|------|
| 主机名 | 设备的主机名（如 `my-server.local`） |
| 系统平台 | 操作系统平台（如 `linux`、`darwin`） |
| 运行时间 | MSM 服务的连续运行时长 |
| 操作系统 | 操作系统类型 |
| 架构 | CPU 架构（如 `amd64`、`arm64`） |

## 硬件信息卡片

展示服务器的硬件配置详情：

| 字段 | 说明 |
|------|------|
| 硬件信息 | CPU 型号名称 |
| 核心数 | CPU 逻辑核心数 |
| 内存 | 物理内存总量 |
| 硬盘容量 | 磁盘总容量 |
| 硬盘使用 | 磁盘已使用百分比（含进度条） |

## 资源使用趋势

实时折线图展示 CPU 和内存的使用趋势：

- 点击 **CPU** / **内存** 按钮切换显示指标
- 支持多种时间粒度：**全部** / **1/2** / **1/4** / **1/5** / **1/10**
- 可设置告警阈值线（如 75%）

::: tip 性能监控建议
如果 CPU 或内存长期超过 80%，建议检查是否有异常进程，或考虑升级硬件配置。
:::

## 实时速率图表

展示网络流量的实时曲线：

- **连接数** — 当前活跃的代理连接数量
- **上传速度** — 实时上传速率（绿色曲线）
- **下载速度** — 实时下载速率（蓝色曲线）
- 支持与资源趋势相同的时间粒度切换

## 统计信息卡片

汇总展示关键运营数据：

| 指标 | 说明 |
|------|------|
| 系统运行时间 | MSM 服务的总运行时长 |
| 总上传流量 | 自启动以来的累计上传数据量 |
| 总下载流量 | 自启动以来的累计下载数据量 |
| CPU 使用率 | 当前 CPU 占用百分比 |
| 内存使用率 | 当前内存占用百分比 |

## 自定义布局

仪表盘支持拖拽自定义布局 <Badge type="tip" text="Pro" />：

1. 点击页面右下角的 **自定义布局** 按钮
2. 拖拽各个卡片到你喜欢的位置
3. 调整完成后布局会自动保存
4. 下次登录时自动恢复你的布局

::: info Pro 功能
自定义仪表盘布局需要 Pro 授权。未激活时使用默认布局。
:::

## 服务状态快捷操作

在仪表盘下方可以看到各服务的运行状态卡片（DNS 服务、代理核心），显示运行状态、PID 和运行时间，支持快捷操作：

- **停止** — 快速停止对应服务
- **重启** — 快速重启对应服务

::: warning 操作提示
停止代理服务（Clash / Sing-Box）会导致所有通过代理的网络连接中断，请确认后再操作。
:::
